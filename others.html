Gitについて

upload:2014.03.30

Gitってご存知ですか?
「ジット」ではなく「ギット」と読みます。

GitHub ←GitHub（ギットハブ）の公式HPはこちらです

Gitは一言でいうと、
作成しているプログラムなどのバージョンを管理する仕組みです。　

難しい言葉では、VCS:Version Control System 日本語では、
バージョン管理システム（難しくなかったですね....）
と呼ばれるものです。

プログラムなどのバージョン管理と言いましたが、
別にプログラムじゃなくても構いません。

たとえばPCで絵を描く人なら、その製作過程を記録しておくことができます。
PCで絵を描く過程が、

下絵　→　下絵の修正　→　着色1　→　着色2　→　微調整　→　完成

だとしましょう。

この「下絵」や「着色2」など、それぞれの過程におけるデータを
Gitでは保存しておくことができます。
（実際には全部保存している訳ではないのですが、そのように扱うことができます）

しかもデータを保存したフォルダをGitに登録しておけば
半自動的に変更履歴を保存してくれます。
（なぜ「半」自動的なのかは次回の投稿で記載しますね）

ですから、手元のデータがたとえ「微調整」の段階に入っていても、
「下絵」や「着色1」など
データを以前の状態まで戻すことができるのです。

しかも、いつ、誰が、何を、どのように変更したのかが確認できるのです。

ここで「誰が」という言葉が出てきて気づいた方もいらっしゃるでしょう。
そう、Gitでは
複数の人でひとつの作品を作ることも可能なのです。

ということは、もちろん

ファイルの共有も可能です。

かなり使い勝手の良いシステムなので世界中で多くの人々が使っています。
実際、僕もこのHPのデータ管理をGitで行っています。

素晴らしいでしょ、Git。

あなたも使ってみませんか?

近日中にGitの導入法とその簡単な使い方について記事を投稿しますのでお楽しみに♪
